## I. This design avoids boolean gates in derivation.

T is the operator. C(T) is the only criterion. Rather than
using if/else to select which path to derive, the streaming
evaluator scores all candidates — the lowest wins. Poorly
performing operators dissolve through score, not through gates.

**Why:** Boolean gates prescribe the answer. The algebra
discovers it. In our experience, every hardcoded parity check,
every intent-gated composition, every puzzle-specific operator
was reverted because it worked against this principle.

**How to apply:** When tempted to write
`if condition { use_this_generator }`, consider instead letting all
generators run and letting C(T) select. Prefer match, Option
dissolution, .then().

Benchmarks test the kernel — they do not define it.

## VII. Observe before executing.

This principle suggests reading live system state before
